AI が検出する3つのフィルター

🔴
赤：最優先アラート（バッドニュース）
経営者が今すぐ知るべき事実
検出条件
  • ▸「クレーム」「怒られた」等の顧客トラブル
  • ▸「遅れている」「失敗した」等の業務トラブル
  • ▸火種・厳しい指摘が含まれる事実の記述
CoSコメントの方向性
大至急の事実確認と対応を促す。「いつ・誰が・どう対応するか」を問う。
🔵
青：100%当事者の欠如（他責・行動停止）
COMPASSの当事者原則からの逸脱
検出条件
  • ▸主語が「自分」ではない他責の文脈
  • ▸「気をつけます」「意識します」等の精神論締め
  • ▸Who / When / What の「次の一手」が不在
CoSコメントの方向性
具体的な行動（What）を引き出す問いかけ。「あなたが明日何をするか」を問う。
🟡
黄：事実と解釈の混同（思考の解像度低下）
成長の余地を示すフラグ
検出条件
  • ▸「事実」と「感情・推測」が混在した文章
  • ▸「何が起きたか」の羅列のみで考察がない
  • ▸根拠のない推測・「〜だと思います」の多用
CoSコメントの方向性
事実と解釈を分離させる問いかけ。「それは事実ですか？あなたの解釈ですか？」

処理フロー（6ステップ）

1
データ取得 セッション認証 CSV ダウンロード

まずID/パスワードでサイボウズOfficeにログインしてセッションCookieを取得。そのCookieを使ってカスタムアプリの前日分CSVをダウンロードし、投稿者名・部署・日付・本文を抽出する。

① フォームPOST → Set-Cookie取得 → ② Cookie付きGET → CSV取得 → ③ CSVパース → 配列変換
2
AI 解析 Gemini 2.5 Flash

全日報テキストとCoSプロンプトをGemini APIへ送信。3つのフィルター条件で逸脱箇所を抽出させる。

3
データ構造化 JSON パース

GeminiのレスポンスをJSONとして受け取り、level / name / extracted_text / cos_comment の形式に変換。

4
メール生成 HTML テンプレート

解析結果を元に赤・青・黄の色分けカードを持つHTMLレポートメールを生成。サマリーバッジも自動計算。

5
メール送信 GmailApp

GASの GmailApp.sendEmail() で社長宛てにHTMLメールを送信。エラー時は自動でエラー通知メールを配信。

6
定期実行 タイムドリブントリガー

GASのトリガーで毎日 07:00〜08:00（JST）に自動起動。 createDailyTrigger() を一度だけ手動実行して登録する。

構築手順（6ステップ）

0

ChromeのDevToolsでURLを確認する ← GASコード実装前に必須

GASがサイボウズOfficeにアクセスするための「CSVダウンロードURL」と「アプリID」を実際のブラウザ操作で確認します。

  1. 手順1 Chromeでサイボウズ Officeを開き、F12キーでDevToolsを起動 → 「Network」タブをクリック
  2. 手順2 日報のカスタムアプリを開き、「CSVに書き出す」を実行する
  3. 手順3 Networkタブに出てきたリクエストを探し「Request URL」をコピーする
  4. 手順4 URLに含まれる pid= の数字が アプリID（CONFIGに設定）
// DevToolsでコピーされるURLのイメージ（例）
https://xxxxxx.cybozu.com/o/a.cgi?fnc=Download&pid=123&charset=UTF-8&type=csv
// ↑ このURLをまるごとコピーしてCONFIGのCSVダウンロード設定に使用
// ↑ pid=123 の数字が CYBOZU_APP_ID

⚠️ ログインURLは https://xxxxxx.cybozu.com/o/ の形が多いですが、環境によって異なります。同様にNetworkタブで確認してください。

A

GASプロジェクトを作成する

  1. 1.script.google.com を開き「新しいプロジェクト」を作成
  2. 2.nippou-alert-system.gs の内容をエディタに貼り付け
  3. 3.プロジェクト名を「日報AIアラートシステム」等に変更
B

スクリプトプロパティに機密情報を保存

GASエディタ → 「プロジェクトの設定」→「スクリプトプロパティ」に以下を追加

キー 値
CYBOZU_USER サイボウズのログインID
CYBOZU_PASSWORD サイボウズのパスワード
GEMINI_API_KEY AIza…（Gemini APIキー）

⚠️ Gemini APIキーは aistudio.google.com で無料発行できます

C

CONFIG の値を自社環境に書き換える

// ▼ ステップ0で確認した値を設定する
CYBOZU_SUBDOMAIN: 'naganetsu', // サブドメイン（URLバーで確認）
CYBOZU_APP_ID: '???', // DevToolsでpid=の数字を確認
// ▼ CSV列番号（実際のCSVで確認済み ✅）
CSV_COLUMNS: {
date: 0, // 日報日付
name: 1, // 氏名
department: 2, // 所属部署
body: 3, // 本日の業務内容
notes: 4, // 気づき（AIに渡して解析精度向上）
tomorrow: 5, // 明日の予定（同上）
},
GEMINI_MODEL: 'gemini-2.5-flash', // 2026年4月推奨
RECIPIENT_EMAIL: 'shacho@naganetsu.co.jp',

✅ CSV列番号は手動ダウンロードで確認済み。CYBOZU_APP_ID のみDevToolsで確認が必要です。

D

testRun() で動作確認

  1. 1.GASエディタのドロップダウンで testRun を選択 → ▶ 実行（初回は権限承認が求められます）
  2. 2.ログに [LOGIN] ログイン成功（JSESSIONID取得済み） と表示されればログイン成功
  3. 3.続けてログに「取得件数 / 解析結果 / HTML文字数」が表示されれば全工程成功
  4. 4.[WARN] _REQUEST_TOKEN_が見つかりません が出ても、その後ログイン成功であれば問題なし
E

createDailyTrigger() でトリガー登録（最後に1回だけ）

  1. 1.ドロップダウンで createDailyTrigger を選択 → ▶ 実行
  2. 2.「トリガー」メニュー（時計アイコン）で登録を確認
  3. 3.翌朝8時台に社長メールアドレスへ届けば完成 🎉
✅ これで毎朝、全日報が自動解析されてアラートメールが届きます

注意事項とカスタマイズポイント

セッション認証・CSVの注意点
  • ・サイボウズ側のUI更新でCSV書き出しURLが変わった場合、スクリプトの修正が必要になります
  • ・SAML/SSO（Googleログイン等）に切り替えると、このセッション認証方式は動作しなくなります
  • ・社員数が多い場合はCSVサイズに注意。GASの実行時間上限（6分）を超えないか確認してください
トリガーのタイミング
  • ・GASのトリガーは指定時間帯に±数分の誤差があります
  • ・確実に8時台に届けるため atHour(7) を設定済み
  • ・土日・祝日もデフォルトで実行されます（要件に応じて曜日フィルター追加可）
Geminiプロンプトの調整
  • ・analyzeWithGemini() 内のsystemPromptを自社のCOMPASS定義に合わせて書き換えられます
  • ・temperature を 0〜0.3 に保つと解析の一貫性が向上します
